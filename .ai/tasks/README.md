# Tasks

Execution tracking for requirement delivery.

## Structure

- `active/current.md` - current working TODO list (single source for in-progress work)
- `backlog/README.md` - deferred items
- `done/README.md` - completed milestones

## Usage

- Break requirements into actionable items.
- Keep exactly one item marked as `IN_PROGRESS`.
- Move completed items into `done` snapshots periodically.
